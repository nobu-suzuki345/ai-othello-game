# 🎮 AI対戦オセロゲーム

JavaScript実装のミニマックス法によるオセロAI

## 📝 概要

このプロジェクトは、Qiita投稿用に作成されたAI対戦オセロゲームです。HTML/CSS/JavaScriptのみで実装されており、ブラウザ上で完結して動作します。

### 特徴

- ✅ **3段階の難易度**
  - 初級：ランダムAI
  - 中級：評価関数ベースAI（貪欲法）
  - 上級：ミニマックス法 + アルファベータ枝刈り

- 🎨 **モダンなUI/UX**
  - レスポンシブデザイン
  - 滑らかなアニメーション効果
  - 配置可能な場所のヒント表示

- 🧠 **高度なAI実装**
  - 位置評価（角を最優先）
  - 機動力評価（有効な手の数）
  - ゲームフェーズ別の戦略

## 🚀 使い方

### 起動方法

1. `index.html` をダブルクリック
2. ブラウザで自動的に開きます
3. すぐにプレイ可能！

**インターネット接続は不要です**

### 操作方法

1. **石を置く**: 緑色で表示される有効な位置をクリック
2. **難易度変更**: 画面左のドロップダウンから選択
3. **新しいゲーム**: 「新しいゲーム」ボタンをクリック
4. **1手戻す**: 「1手戻す」ボタンでプレイヤーの手を取り消し
5. **ヒント**: 「ヒント」ボタンで推奨手を表示

## 🧠 AI実装の詳細

### 初級AI（難易度1）
```javascript
// ランダムに手を選択
selectRandomMove(validMoves)
```
- 有効な手からランダムに選択
- 勝つのが簡単

### 中級AI（難易度2）
```javascript
// 評価関数ベース
selectGreedyMove(board, validMoves, player, game)
```
- 位置評価値を使用
- 角（100点）、辺（10点）を優先
- X位置とC位置を避ける（-20〜-50点）

### 上級AI（難易度3）
```javascript
// ミニマックス法 + アルファベータ枝刈り
selectMinimaxMove(board, player, game)
```

**探索深度**: 4手先まで読む

**評価関数の要素**:
1. **位置評価** - 角や辺の価値を評価
2. **石数評価** - 終盤で石の数を重視
3. **機動力評価** - 序盤〜中盤で有効な手の数を評価
4. **角の確保** - 角の重要性を強調（+100/-100点）

**最適化技術**:
- アルファベータ枝刈りで探索効率を向上
- ゲームフェーズ別の評価戦略

## 🎯 評価関数の位置評価表

```
  A   B   C   D   E   F   G   H
1 [100][-20][ 10][  5][  5][ 10][-20][100]
2 [-20][-50][ -2][ -2][ -2][ -2][-50][-20]
3 [ 10][ -2][  5][  1][  1][  5][ -2][ 10]
4 [  5][ -2][  1][  0][  0][  1][ -2][  5]
5 [  5][ -2][  1][  0][  0][  1][ -2][  5]
6 [ 10][ -2][  5][  1][  1][  5][ -2][ 10]
7 [-20][-50][ -2][ -2][ -2][ -2][-50][-20]
8 [100][-20][ 10][  5][  5][ 10][-20][100]
```

- **角（100）**: 最優先で確保
- **角の隣（-20〜-50）**: 相手に角を取られるリスク
- **辺（10）**: 安定した位置
- **中央（0〜5）**: 柔軟性がある位置

## 🔧 カスタマイズ

### AI難易度の調整

`ai.js` の `maxDepth` を変更することで、上級AIの強さを調整できます：

```javascript
this.maxDepth = 4;  // デフォルト（4手先読み）
```

- `maxDepth = 2`: 弱い（高速）
- `maxDepth = 4`: 標準（推奨）
- `maxDepth = 6`: 強い（遅い）

### 位置評価値のカスタマイズ

`ai.js` の `positionWeights` 配列を編集することで、AIの戦略を変更できます。

## 🎓 学習リソース

### オセロのルール
- 8×8の盤面で黒と白の石を使用
- 相手の石を挟むと裏返る
- 最後に石が多い方が勝ち

### アルゴリズム
- **ミニマックス法**: ゲーム木を探索する基本アルゴリズム
- **アルファベータ枝刈り**: 探索効率を向上させる最適化
- **評価関数**: 局面の良し悪しを数値化
